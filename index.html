Parallel Graph Coloring

Advanced Parallel Implementation of Graph Coloring Algorithms

OpenMP MPI C/C++ Transactional Memory

Andrew Id: svattiku, balasubs

Summary

Our final project will implement and analyze multiple parallel approaches to the graph coloring problem, with a novel focus on a transactional memory-inspired approach using optimistic execution and conflict resolution.

Background

Graph coloring assigns colors to nodes such that no adjacent nodes share the same color. This problem has critical applications in register allocation, scheduling, and frequency assignment in wireless networks. The minimum graph coloring problem is NP-Complete, necessitating approximation algorithms.

The greedy approach is a common approximation algorithm, assigning the lowest possible color to each vertex while avoiding conflicts with already-colored neighbors. However, this sequential method becomes increasingly slow for large graphs.

Parallelization Approaches

Independent Set Coloring

Identifies sets of vertices that can be colored simultaneously without conflicts. Implemented using OpenMP for thread-level parallelism and MPI for distributed identification and coloring.

Domain Decomposition

Partitions the graph into subgraphs and colors them in parallel, with special handling for boundary vertices using OpenMP and MPI.

Transactional Memory-Inspired Approach

Innovative approach using optimistic parallel coloring with post-coloring conflict detection and resolution, mimicking transactional memory systems.

Novel Transactional Memory-Inspired Approach
Key Characteristics
  • Concept: Optimistically color vertices in parallel without initial synchronization
  • Conflict Detection: Parallel scan to identify adjacent vertices with same color
  • Resolution Strategies:
    • Priority-based resolution
    • Timestamp ordering
    • Probabilistic resolution with exponential backoff

Unlike traditional approaches that prevent conflicts through synchronization, this method allows conflicts and resolves them afterward, enabling more independent thread operation.

Challenges

Load Balancing

Addressing non-uniform graph structures to distribute work evenly among processors, especially for irregular graphs.

Dependency Management

Managing vertex dependencies to prevent race conditions where neighboring vertices receive the same color.

Performance-Quality Tradeoffs

Balancing parallel implementation efficiency with color count minimization and speedup maintenance.

Additional Specific Challenges
  • Balancing optimistic coloring with conflict resolution overhead
  • Developing scalable conflict detection mechanisms
  • Controlling color count during resolution processes

Resources & Platform Choice

Our implementation will start with a C/C++ sequential greedy graph coloring algorithm, which will be extended to parallel versions. Initial development will occur on GHC machines, with comprehensive performance evaluation planned on PSC machines with superior core counts.

We selected C/C++ based on prior experience with OpenMP and OpenMPI, frameworks well-suited to our parallel approaches.
